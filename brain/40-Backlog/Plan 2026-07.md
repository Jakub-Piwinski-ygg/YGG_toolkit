(`SceneStudioInner.jsx:784` flow/scenario state, `PixiViewport.jsx:196`
render + Spine tick, `viewportController.js:276` drag) and only the middle one
honors `exportingRef` — the shared root of bugs B1 and B3.

## Stage 0 — Bugs (P0) ✅ DONE
- **B1 Export vs playback fight** — WebM export suppresses its own loop but the
  SceneStudioInner state RAF keeps ticking `flowState` → competing
  `applyFlowAtTime` + `app.render()` (PixiViewport cheap path :337-363).
  Fix: one `exportActive/playbackSuspended` flag honored by all three loops;
  opening the export dialog stops playback. (S/M)
- **B2 Number inputs clamp mid-typing** — `DragNumberField.jsx:104-109` calls
  `onChange(clamp(v))` per keystroke; dozens of bare `<input type="number">`
  with inline clamps app-wide (wizards, TimelinePanel, CheatTool, art tools).
  Fix: commit-on-Enter/blur in DragNumberField; new shared `NumberField`;
  migrate all call sites. Rule added to CLAUDE.md + [[Gotchas]] ✅. (M)
- **B3 Gizmo intermittently dead** — causes: `getObjectMetrics` null on 0×0
  bounds (Spine mid-load, pixiApp.js:479); rebuild window (:295-332);
  selection forced null in direct mode/wizard (:3386); playback's
  `syncTransforms` overwrites in-progress drags. Fix: bounds fallback cache,
  drag priority over sync, diagnostics to app log. Shares B1's flag. (M)
